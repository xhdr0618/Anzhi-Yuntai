# 安置云台

## Commit 规范

我们采用 [Conventional Commits](https://www.conventionalcommits.org/zh-hans/v1.0.0/) 规范来确保 commit 信息的一致性。

### Commit 消息格式

```markdown
<type>[optional scope]: <description>

[optional body]

[optional footer]
```

### Commit 类型

- `feat`: 新功能
- `fix`: 修复 bug
- `docs`: 文档更新
- `style`: 代码格式化
- `refactor`: 代码重构
- `test`: 测试
- `chore`: 其他

### Commit 示例

```markdown
fix(button): 修复按钮点击事件
```
